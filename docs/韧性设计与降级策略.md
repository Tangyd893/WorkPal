# 韧性设计与降级策略

## 消息发送幂等

前端发送消息时生成 `idempotency_key`，后端同时支持请求体字段和 `Idempotency-Key` 请求头。

后端策略：

- 以 `conv_id`、`sender_id`、`idempotency_key` 组合去重。
- 幂等键有效期为 5 分钟。
- 过期幂等键会在下一次发送时清理，避免同一客户端长期无法复用令牌。
- 数据库增加唯一索引，降低并发重复提交风险。

## 搜索服务降级

搜索不可用时：

- 前端展示“搜索暂不可用，消息收发不受影响。”
- 搜索结果区清空并保持聊天输入可用。
- Gateway 对搜索服务单独使用 5 次失败、30 秒冷却的熔断策略。

## 网关韧性

Gateway 已具备：

- 限流
- 重试
- 熔断
- 服务注册发现
- 健康检查
- 上游服务响应头标识

## 回归验证

- 重复提交同一个 `idempotency_key`，应返回同一条消息。
- 停止搜索服务后执行消息搜索，前端应只提示搜索降级，不影响发送消息。
- 连续触发搜索服务 5 次 5xx 后，Gateway 应打开熔断并在 30 秒后进入半开。
