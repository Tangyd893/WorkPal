# WorkPal 前端组件架构说明

## 架构目标

本次优化把原先单体的 `WorkspacePage` 收敛为路由编排层，页面自身控制在 200 行以内，具体视图、数据加载、业务操作和反馈组件分别下沉到独立模块。这样可以让后续新增模块时只扩展局部文件，不再继续堆高工作台入口页。

## 当前组件树

```text
App
  ├─ ErrorBoundary
  └─ Layout
     └─ WorkspacePage
        ├─ Sidebar
        ├─ Topbar
        ├─ WorkspaceContent
        │  ├─ OverviewPanel
        │  ├─ ChatPage
        │  ├─ TasksPanel
        │  ├─ SchedulePanel
        │  ├─ FilesPanel
        │  └─ DirectoryPanel
        ├─ SettingsDrawer
        ├─ ModuleSwitcher
        ├─ ConfirmDialog
        └─ ToastViewport
```

## 数据与操作分层

- `useWorkspaceData`：负责当前用户、团队成员、部门、任务、日程、文件和通讯录搜索状态。基础资料先加载，任务、日程、文件按当前模块懒加载；通讯录搜索使用 `AbortController` 与请求序号避免旧响应覆盖新结果。
- `useWorkspaceActions`：封装任务、日程、文件的创建、状态更新、分享、删除和上传逻辑。操作结果统一进入 Toast，不再把一次性反馈堆在页面顶部。
- `WorkspaceContent`：只负责根据 `activeSection` 选择模块组件，并把确认弹窗请求交还给页面。
- `stores/authStore.ts`、`stores/wsStore.ts`、`stores/convStore.ts`：认证、WebSocket 消息和会话状态已从原聚合 store 拆分，旧 `hooks/useAuthStore.ts` 保留再导出以兼容现有引用。

## 代码分割

六个工作台模块通过 `React.lazy` 加载，`WorkspacePage` 使用 `Suspense` 提供骨架占位。当前进入某个模块时只加载对应组件代码，概览页作为聚合入口会触发任务、日程、文件三类数据加载。

## 错误兜底

`ErrorBoundary` 已接入两个层级：

- App 顶层：避免根组件异常造成白屏。
- 工作台内容区：单个模块渲染失败时显示可重试降级界面，不影响侧边栏、顶部栏和偏好设置。

## 路由增强

- `/workspace/:section` 保持原有模块路由。
- `/workspace/chat/:conversationId` 支持会话直达和分享。
- 未匹配地址进入 `NotFoundPage`，不再全部静默重定向。
