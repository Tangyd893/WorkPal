# 工作上下文智能检索与洞察平台设计

## 目标

把任务、消息、文件、日程等工作上下文统一进入可检索、可聚合、可分析的数据平台。

## 设计方案

- 数据入口：Debezium 或业务 outbox 监听多表变更。
- 消息总线：Kafka 承载跨服务变更事件。
- 索引层：Elasticsearch 建立统一工作上下文索引。
- 查询层：支持关键词、时间范围、实体类型、状态、参与者组合过滤。
- 洞察层：基于 Elasticsearch 聚合分析项目参与度、投入时间占比、协作活跃度。

## MVP 计划

第一周：

- 梳理事件模型。
- 建立数据同步管道。
- 建立基础索引和基础查询接口。

第二周：

- 增加自然语言查询解析。
- 增加聚合报表。
- 增加权限过滤与审计。

## 与当前仓库的衔接

当前 Search Service 已经具备 Bleve、outbox、Redis Streams 链路，可作为 MVP 的轻量版基础。后续升级 Elasticsearch 时，可以复用消息事件和权限校验边界。
