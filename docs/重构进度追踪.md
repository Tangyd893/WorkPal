## 待推进事项

### 搜索主线（P2-8）
- 当前：Search Service 使用 Bleve 索引，ES 仅为容器预留
- 方案 A：继续 Bleve，移除设计文档中"已完成 Elasticsearch 搜索"的表述
- 方案 B：迁移到 Elasticsearch，实现索引 mapping、批量写入、搜索高亮
- **当前决策**：继续 Bleve 为主线，ES 作为后期升级备选

### 事件总线（P2-9）
- 当前：业务主链路使用 Redis Streams，Kafka/Redpanda 容器已就绪
- 方案 A：继续 Redis Streams，清理 Kafka 相关文档中的过度声明
- 方案 B：迁移到 Kafka，接入 IM outbox、Workspace Saga、通知事件
- **当前决策**：继续 Redis Streams 为主线，Redpanda 作为后期大规模场景备选

### 协作文档升级（P2-10）
- 当前：文档 CRUD + revision，前端 textarea 编辑
- 待做：引入 TipTap/ProseMirror 编辑器，Yjs CRDT 协同，版本历史/冲突处理

### AI 真实接入（P2-11）
- 当前：noopSearch 占位，简单字符串拼接
- 待做：接入 Search Service 检索，LLM provider 抽象（超时/重试/审计/脱敏）

### 日历会议室升级（P2-12）
- 当前：会议室为进程内存 map
- 待做：WebRTC signaling API，前端入会/离会/静音/屏幕共享交互

## 可验证标准

| 模块 | 验证方式 |
|------|---------|
| Project Service | 项目创建 → 默认 Issue 类型自动初始化 → Issue 创建成功 |
| IM Channel | 创建频道 → 重启服务 → 频道数据仍存在 |
| Notification | 后端 API 返回通知列表 → 前端 Topbar 展示真实通知数据 |
| CustomField | 空库启动 project-service 不报错 → 自定义字段 API 创建/写入/读取闭环 |
| Compose 启动 | `docker compose up -d` 启动所有 11 个微服务，Gateway /services 可查 |

## 当前可用验证命令

```powershell
cd frontend
npm run lint    # ESLint 检查
npm test        # 60 个测试全部通过
npm run build   # TypeScript + Vite 构建成功
```
