# 生产部署指南

## 部署资产

- 后端多阶段构建镜像：`backend/Dockerfile`
- 本地完整栈：`docker/docker-compose.yaml`
- 生产配置模板：`backend/configs/生产配置模板.yaml`
- Kubernetes 工作负载：`docker/kubernetes/工作负载.yaml`
- 监控配置：`docker/监控/`

## 配置原则

配置优先级为：环境变量覆盖 `config.yaml`，敏感项必须从环境变量或 Kubernetes Secret 注入。

必须替换：

- `SERVER_JWTSECRET`
- `SERVER_INTERNALTOKEN`
- `DATABASE_PASSWORD`
- `REDIS_PASSWORD`
- `FILE_MINIO_ACCESSKEY`
- `FILE_MINIO_SECRETKEY`
- Grafana、MinIO、数据库控制台默认口令

## Docker Compose

```bash
docker compose -f docker/docker-compose.yaml build
docker compose -f docker/docker-compose.yaml up -d
```

启动后可访问：

- 网关：`http://localhost:8080`
- Prometheus：`http://localhost:9090`
- Grafana：`http://localhost:3001`
- Jaeger：`http://localhost:16686`

## Kubernetes

```bash
kubectl apply -f docker/kubernetes/工作负载.yaml
kubectl -n workpal get pods
kubectl -n workpal get svc
```

上线前建议将镜像名 `workpal/<service>:latest` 替换为带版本号的镜像，例如 `workpal/gateway:2026.05.05`。

## 健康检查

- 网关：`/health/live`、`/health/ready`
- 业务服务：`/health`
- 指标端点：`/metrics`

Kubernetes 清单已经为 6 个微服务配置 readinessProbe、livenessProbe、资源 requests/limits 和滚动更新策略。

## 回滚建议

1. 使用不可变镜像标签发布。
2. 保留上一版本 Deployment。
3. 异常时执行：

```bash
kubectl -n workpal rollout undo deployment/gateway
```

4. 观察 `/health/ready`、Grafana 错误率和 P99 延迟。
