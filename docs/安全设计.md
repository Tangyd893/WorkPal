# 安全设计

## 审计日志

各服务新增 `audit_logs` 表，字段包括：

| 字段 | 说明 |
|:---|:---|
| `user_id` | 操作用户 |
| `action` | 操作名称 |
| `resource_type` | 资源类型 |
| `resource_id` | 资源 ID |
| `ip` | 客户端 IP |
| `timestamp` | 操作时间 |

已记录的敏感操作：

- 删除消息
- 移出群成员
- 删除会话
- 删除任务
- 删除日程
- 下载文件
- 删除文件

## 敏感配置保护

仓库 `.gitignore` 已覆盖：

- `.env`
- `backend/configs/config.yaml`
- `*.log`
- 本地上传和搜索数据目录

生产环境必须从 Secret 或环境变量注入敏感配置，不允许写入仓库。

## 上线检查清单

- 替换 JWT 密钥。
- 替换服务间调用密钥。
- 替换数据库、Redis、MinIO 密码。
- 禁用或保护默认控制台口令。
- 检查公网只暴露 Gateway 和必要控制台。
- 开启审计日志归档策略。
- 定期检查 `audit_logs` 中的异常删除、下载、踢人操作。
